## 一、产品概述

### 1.1 产品名称
**TradeMind AI** · AI 交易时空复盘与行为教练

### 1.2 一句话定位
为港美股/A 股个人投资者提供"B/S 点时空还原 + AI 行为画像 + 胜率归因"的智能复盘工具，帮助散户从"盲目赌博"进化为"系统性交易"。

### 1.3 目标用户
- **主要用户**：具备一定交易经验、收益不稳定、渴望进阶的"进取型"散户
- **服务渠道**：券商平台内嵌 H5 应用（微信/App 内打开）
- **覆盖市场**：A 股（当前 MVP）；港美股（后续迭代）

### 1.4 商业价值
| 对象 | 价值 |
|---|---|
| 用户 | 提供机构级交易归因分析，克服人性弱点，提升投资能力 |
| 券商/平台 | 增强用户留存与粘性（深度数据依赖），促活转化投教产品 |

---

## 二、核心痛点

| # | 痛点 | 描述 |
|---|---|---|
| 1 | **只知盈亏，不知归因** | 传统券商 App 只提供买卖点和盈亏数字，用户无法回溯当时的决策动机（FOMO 追高、恐慌抛售等） |
| 2 | **缺乏深度交易归因** | 胜率统计是结果导向的，用户不知道亏损是因为财报季方向失败还是纯粹情绪化追高 |
| 3 | **复盘门槛极高** | 还原一笔交易当时的宏观环境、个股异动、新闻情绪需耗费大量时间，普通人无法坚持 |

### 模块 1：AI 交易时空机（Time-Machine Replay）

**功能描述**：当用户点击历史交割单上某个买入/卖出点时，AI 自动抓取该时间戳（精确到分钟级）前后的核心市场切片。

**AI 介入点**：运用 RAG（检索增强生成）技术，检索该时间点的个股突发新闻、推特/雪球等社交情绪指数、大盘异动情况，生成一句话总结。

**MVP 当前实现状态**：
- ✅ 历史 K 线展示（买卖点标记，绿色▲买入 / 红色▽卖出）
- ✅ ECharts 蜡烛图，含买卖前后 10 日窗口
- ✅ 行情数据来源：QVeris 实时 API + Supabase 本地缓存
- ⬜ RAG 新闻检索（待开发）
- ⬜ 社交情绪指数接入（待开发）

---

### 模块 2：AI 交易行为诊断

**功能描述**：结合用户长期的 B/S 点与时空机数据，AI 为用户生成动态的交易心理与行为画像。

**当前识别的 5 种核心交易模式**：

| 模式 | 描述 | 识别方式 |
|---|---|---|
| 追高买入（chase_high） | 在涨幅 >3% 时买入 | 规则算法 |
| 止盈过早（early_profit） | 持仓未达潜在空间即卖出 | 规则算法 |
| 止损过慢（slow_stop_loss） | 亏损超过阈值仍未止损 | 规则算法 |
| 频繁交易（over_trading） | 单周交易次数异常偏高 | 规则算法 |
| 持仓过久（hold_too_long） | 持仓天数远超同类股票均值 | 规则算法 |

**数据挖掘维度**（规划中）：
- 新闻敏感度：左侧交易者 vs 右侧交易者
- 情绪抵抗力：市场恐慌指数（VIX）飙升时的胜率
- 持仓耐心值：盈利单与亏损单的平均持仓周期对比

**MVP 当前实现状态**：
- ✅ 5 种模式规则检测
- ✅ 严重程度评分（0-100）
- ✅ 数据预警 Banner（< 10 笔为"样本不足"，< 30 笔为"初步结论"）
- ✅ LLM 生成交易风格描述（`ai_style_description`）
- ⬜ 新闻敏感度 / 情绪维度分析（待开发）

---

### 模块 3：AI 胜率归因与策略教练

**功能描述**：基于行为画像，AI 提供个性化的"避坑指南"和"胜率提升建议"。

**AI 输出内容**：
- 复盘总结（`ai_summary`）
- 改进建议列表（`ai_suggestions`）
- 各模式典型案例点评（`ai_commentary`）

**回测模拟**（当前支持 3 种策略）：

| 策略 | 描述 |
|---|---|
| 止损线下调 2% | 模拟更严格止损带来的收益改善 |
| 止盈线上调 5% | 模拟更充分持仓带来的收益改善 |
| 减少追高 | 模拟剔除高位追入交易后的净收益变化 |

**可视化**：原始策略 vs 最优策略的累计盈亏折线图

**MVP 当前实现状态**：
- ✅ 3 种回测场景
- ✅ 累计盈亏折线图（ECharts）
- ✅ LLM 生成建议文本
- ⬜ 更多策略场景（待扩展）
- ⬜ 用户自定义参数（待开发）

---

### 模块 4：T+N 决策回放卡片（规划中）

**功能描述**：在交易发生后的 T+7 或 T+30，系统自动推送一张复盘卡片。

**输出示例**：
> "您在 30 天前以 180 美元买入 AAPL。当时该股 RSI 达 78 处于超买区间，且市场正处于极度贪婪状态。此后 30 天股价回调 6.2%。回顾来看，这笔交易受 FOMO 情绪驱动明显。建议下次遇到高位缩量，多观察 3 个交易日。"

**MVP 当前实现状态**：
- ⬜ 全部待开发

## 四、技术架构

### 4.1 当前部署架构

```
用户（手机浏览器）
        │
        ▼
Cloudflare Pages          ← 前端 React H5（全球 CDN）
https://fincoach-aee.pages.dev
        │
        ├──▶ Render（FastAPI）  ← 分析引擎 + API
        │    https://fincoach-backend.onrender.com
        │
        └──▶ Supabase PostgreSQL  ← 数据库
             aws-ap-northeast-1（东京）
```

### 4.2 技术栈

| 层 | 技术 | 说明 |
|---|---|---|
| 前端 | React 18 + TypeScript + Vite | H5 移动端适配 |
| UI 组件 | Ant Design Mobile | 移动端组件库 |
| 图表 | ECharts | K 线、盈亏图、回测图 |
| 后端 | Python 3.11 + FastAPI | 异步 API 服务 |
| 分析引擎 | pandas + numpy | 交易模式规则计算 |
| AI | OpenAI 兼容 API（Claude） | 复盘报告生成 |
| 数据库 | Supabase PostgreSQL 17 | 用户/交易/行情/报告 |
| 行情数据 | QVeris API | 实时 + 历史 K 线 |
| 部署-前端 | Cloudflare Pages | 免费，全球 CDN |
| 部署-后端 | Render | 免费套餐，Python 支持 |

### 4.3 数据库表结构

| 表名 | 说明 | 当前数据量 |
|---|---|---|
| `users` | 用户基本信息 + 画像类型 | 5（mock） |
| `trades` | 交易流水（买卖记录） | 220（mock） |
| `market_data` | 股票历史 K 线 OHLCV | 570（mock） |
| `reports` | 生成的复盘报告（含 JSONB） | 按需增长 |

### 4.4 报告生成流程

```
POST /api/reports/generate
        │
        ├─ 立即返回 report_id（status=generating）
        │
        └─ 后台异步 pipeline：
           Step 1: 读取交易记录（Supabase）
           Step 2: 交易模式识别（规则算法）
           Step 3: 问题诊断 & 评分（规则算法）
           Step 4: 回测模拟（规则算法）
           Step 5: AI 报告生成（LLM）
                   → 写回 Supabase（status=completed）

前端每 3 秒轮询 GET /api/reports/{id}
→ 检测到 completed → 渲染报告
```

## 五、用户旅程（当前 MVP）

```
1. 打开应用
   └─ 首页：选择账户 + 复盘周期

2. 点击「生成复盘报告」
   └─ 进度页：5 步动态进度展示（30～60 秒）

3. 报告页展示：
   ├─ ⚠️ 数据预警 Banner（样本不足时显示）
   ├─ 📊 本期交易风格（LLM 生成）
   ├─ 📈 交易概览（笔数/胜率/盈亏/持仓天数）
   ├─ 📉 盈亏分布图
   ├─ 🔍 交易模式横滑卡片（K 线 + AI 点评）
   ├─ 🩺 诊断结果（严重程度评分 + 主要问题）
   ├─ 🔬 回测对比（原始 vs 最优策略折线图）
   ├─ 🤖 AI 复盘总结
   └─ 💡 改进建议

4. 历史报告列表
   └─ 查看历史各期报告
```
